# 🐍 [2026-01] 나만의 파이썬 소프트웨어 개발 프로젝트

## 1. 시나리오 제목
* 파이썬 포켓몬 마스터 육성 시스템
<br>


## 2. 시나리오 (5~10줄)
* 모험의 시작 (데이터 입력): 프로그램이 시작되면 사용자(트레이너)의 이름을 묻고, 함께 모험을 떠날 첫 포켓몬을 선택받습니다.
트레이너는 야생포켓몬과 조우하기도 하고 다른 트레이너와의 대결을 진행하기도 합니다.

* 등급 판정 (조건문): 계산된 능력치 총합에 따라 S급부터 F급까지 포켓몬의 잠재 등급을 매깁니다.

* 포켓몬 배틀: 야생 포켓몬 또는 다른 트레이너와의 배틀에서 승리하면 돈과 경험치를 얻을 수 있습니다.

* 포켓몬 도감 관리 (리스트/딕셔너리): 한 마리만 관리하는 것이 아니라, 모험 중에 새로 잡거나 만난 포켓몬들을 '포켓몬 도감(리스트)'이라는 커다란 저장소에 차곡차곡 쌓습니다. 사용자는 언제든 도감을 열어 내가 가진 포켓몬들의 목록과 상태를 확인할 수 있습니다.

* 무한 모험 시스템 (반복문 및 메뉴): 사용자가 '모험 종료'를 선택하기 전까지 프로그램은 멈추지 않습니다. 메인 메뉴에서 1. 포켓몬 상태 확인, 2. 계속 진행하기, 3. 도감 보기, 4. 종료와 같은 번호를 입력받아 원하는 기능을 자유롭게 반복해서 수행할 수 있는 완전한 소프트웨어 형태로 완성합니다.

<br>


## 3. 예상 기능 및 메뉴 (최소 5개)
1. 신규 포켓몬 등록
2. 포켓몬 종족값 계산
3. 야생 포켓몬, 트레이너와의 배틀
4. 상성 기반 데미지 시스템 적용
5. 나의 포켓몬 도감 조회
0. 게임 종료

### **🤖 AI 파트너십 과정**

    * **적용 내용:** AI가 제안한 아이디어 중 **'텍스트 기반 포켓몬 배틀 시뮬레이터'**라는 컨셉이 가장 마음에 들어 채택함. 단순히 텍스트 기반 게임 코드를 짜는 게 아니라 포켓몬 배틀을 직접 구현한다는 목표를 세우니 기획이 훨씬 즐거워짐.
      
 2. **내용 2 : 세부 기능 설계**
    * **프롬프트 요약:** "예상 기능 및 메뉴를 생각하고, 포켓몬 배틀 또한 진행할 것인데 어떻게 할 것인가"
    * **적용 내용:** 신규 포켓몬 등록 및 선택, 포켓몬 종족값 설정, 포켓몬 배틀 기능, 상성 기반 데미지 시스템 적용, 나의 포켓몬 도감 확인과 같은 기능들을 추가할 것이고, 포켓몬 배틀과 같은 경우에는 random 라이브러리를 이용하여 배틀의 역동성을 더욱 높일 수 있다.

## 🟩 [2차 과제: V1.0] 입출력 + 리스트 + 조건문 - 향후 작성 예정
### **✨2차 과제 내용:**
  * 이번 과제는 사용자(트레이너) 등록 및 기본적인 시스템 구현으로, 사용자 데이터를 입력받고, 만나는 포켓몬의 데이터를 도감 리스트에 저장하는 시스템을 구축하는 것입니다. 특히 `if-elif` 조건문을 활용하여 사용자의 입력에 따라 프로그램이 각기 다른 기능을 수행하도록 합니다.
  * input()으로 트레이너의 이름을 입력받아 저장.
  * 야생에서 조우한 포켓몬의 정보를 도감 리스트에 저장.
  * if-elif-else를 사용하여 사용자의 입력(menu, choice)에 따라 프로그램이 각기 다른 기능을 수행하도록 분기 처리.
  * random 라이브러리를 이용하여 야생에서 어떤 포켓몬이 나올지, 얼마나 데미지를 입을지 결정.
    
### **🤖 AI 파트너십 과정**
1. **내용 1: 포켓몬 모험 시스템 컨셉 및 로직 도출**
    * **프롬프트 요약**: "내가 포켓몬 트레이너가 되어 첫 포켓몬을 선택하고, 야생 포켓몬을 조우하며 모험하는 프로그램을 만들 거야. 리스트와 조건문을 활용해 5가지 메뉴가 작동하는 로직을 짜줘."
    * **적용 내용**: AI가 제안한 '스타팅 포켓몬 선택 리스트'와 '랜덤 함수를 활용한 야생 포켓몬 조우 시스템'을 반영함. 특히, 야생에서 만난 포켓몬이 도감 리스트에 없을 때만 새로 추가되는 '중복 체크 로직'을 적용하여 실제 게임과 같은 수집의 재미를 구현함.

2. **내용 2: 단계별 포켓몬 프로그램 확장 로드맵 자문**
    * **프롬프트 요약:** "파이썬 기초 문법 단계(리스트 → 조건문 → 반복문 → 함수/딕셔너리)에 맞춰, '포켓몬 트레이너' 프로그램이 어떻게 발전할 수 있을지 4단계 기능 확장 로드맵을 제안해줘."
    * **적용 내용:** 단순 메뉴 선택을 넘어, '기초 포켓몬 선택 및 상태창 구축(1차) → 랜덤 배틀 및 도감 등록 시스템(2차) → While 루프를 활용한 무한 모험 시스템(3차) → 딕셔너리와 함수를 활용한 스킬 시스템 및 다수 포켓몬 보유 기능(4차)'으로 이어지는 체계적인 개발 방향성을 확립함.
    
### **🛠️ Troubleshooting & 기술 회고:** 
1. **문제 1: 메뉴 선택 시 조건문이 작동하지 않는 현상 (자료형 오류)**
    * **원인:** 파이썬의 input() 함수로 받은 값은 숫자가 아닌 **'문자열(String)'**로 저장됨. 즉, 숫자 1과 문자 '1'을 다르게 인식하여 조건문이 False를 반환함.
    * **해결:** menu = int(input())과 같이 **정수형 변환(Type Casting)**을 거치거나, 조건식의 비교 대상을 문자열(if menu == '1':)로 수정하여 데이터 타입의 일관성을 맞춤으로써 해결함.
